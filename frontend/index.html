How does it work?

If you're seeing this message in your browser, it means your Docker Compose cluster is up and running!

The components enumerated in etc/docker-compose.yaml file are fulfilling various roles, akin to a production setup yet overly simplified:

  • freeswitch - a generic FreeSWITCH server, customizable via etc/freeswitch.xml
  • eqivo - the actual Eqivo application, configured via environment variables
  • frontend - substitutes a public website; comes with a simple webRTC webphone to mimic PSTN interactions
  • backend - impersonates a simple application backend, written in academic node.js

These components work together to achieve the goals set for each scenario; their source code is deliberately kept simple (naive) so the emphasis is on the Eqivo integration rather than being opinionated on how an application stack should be built.

The choice of a node.js environment for the backend is driven solely by its (current) popularity, you can use any environment able to send and receive HTTP calls.

Lastly, don't forget to grant the media permissions for your microphone before launching any of the provided scenarios!
